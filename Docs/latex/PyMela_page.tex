\item Functions that are implemented as pass-\/by-\/reference in C++ that need to be converted to functions that return a value

The first two categories of functions are detailed in the \mbox{\hyperlink{group__Pychanges}{Novel Python Functions}} section. \hypertarget{MELA_python}{}\doxysection{M\+E\+LA}\label{MELA_python}
This is what contains the bulk of what one desires to do with M\+E\+LA within Python. The baseplate of this, is of course, the \mbox{\hyperlink{classMela}{Mela}} class in {\ttfamily C++}.\hypertarget{MELA_python_MELA_constructor}{}\doxysubsection{Constructors}\label{MELA_python_MELA_constructor}
The constructor inherits from the constructor in the \mbox{\hyperlink{classMela_MELA_constructor_cpp}{C++ implementation}}. All the default arguments work, so one could create the \mbox{\hyperlink{classMela}{Mela}} object with just a call to {\ttfamily \mbox{\hyperlink{classMela}{Mela()}}}.\hypertarget{MELA_python_MELA_couplings}{}\doxysubsection{Couplings}\label{MELA_python_MELA_couplings}
Couplings in M\+E\+LA are what define the interactions between different particles.

These are couplings using the amplitude basis for the Higgs Boson. These are described in the following papers on the main page for J\+H\+U\+Gen linked \href{https://spin.pha.jhu.edu/}{\texttt{ here}}.

In Mad\+M\+E\+LA (or the Madgraph matrix elements used within M\+E\+LA), the values set will colloquially be called \char`\"{}couplings\char`\"{} but are actually the Wilson Coefficients that are used in Standard Model Effective Field Theory (S\+M\+E\+FT). Under the S\+M\+E\+F\+T\+Sim framework, all the Wilson Coefficients being set to 0 corresponds to the Standard Model. The S\+M\+E\+F\+T\+Sim formulation is described in the following paper \href{https://arxiv.org/abs/2012.11343}{\texttt{ here}}.\hypertarget{MELA_python_autotoc_md2}{}\doxysubsubsection{Standard Model Definition}\label{MELA_python_autotoc_md2}
In M\+E\+LA, the Standard Model is defined as\+:


\begin{DoxyItemize}
\item {\ttfamily ghz1=1, ghg2=1} for J\+H\+U\+Gen
\item {\ttfamily ghz1=1, kappa\+\_\+top=1, kappa\+\_\+bot=1} for M\+C\+FM
\item No couplings set for Mad\+M\+E\+LA (S\+M\+E\+F\+T\+Sim default is Standard Model)
\end{DoxyItemize}\hypertarget{MELA_python_autotoc_md3}{}\doxysubsubsection{Coupling Values}\label{MELA_python_autotoc_md3}
M\+E\+LA in Python relies upon named couplings, rather than couplings done through indices as in {\ttfamily C++}. See the following table of all couplings \mbox{\hyperlink{MELA_couplings_table}{here}}. All of the possible couplings/\+Wilson Coefficients you would want to set are defined by macros in \mbox{\hyperlink{mela__binding_8cpp}{mela\+\_\+binding.\+cpp}}. The definition for those macros can be found \mbox{\hyperlink{group__Macros_py_macros}{here}}.

{\bfseries{Most}} couplings can be both real and imaginary. As a consequence, {\bfseries{most}} couplings are assigned via a Python iterable of size 2.

Some couplings are different. For example, all the S\+M\+E\+F\+T\+Sim couplings for usage within Mad\+M\+E\+LA are single-\/valued real inputs.
