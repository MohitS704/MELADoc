The type {\ttfamily Simple\+Particle\+Collection\+\_\+t} is a typedef described in \mbox{\hyperlink{TVar_8hh}{T\+Var.\+hh}}. It is a vector of {\ttfamily Simple\+Particle\+\_\+t} objects. These correspond to the particle lists that are submitted to \mbox{\hyperlink{classMela_a71142f41d44bb42a1cd2be50b3cc8240}{Mela\+::set\+Input\+Event}}. There are a few constructors for this object, each of which depend on what suits your needs.\hypertarget{SimpleParticleCollection_t_simpleparticlecollection_constructor}{}\doxysection{Constructors}\label{SimpleParticleCollection_t_simpleparticlecollection_constructor}
There are 4 different ways to construct the {\ttfamily Simple\+Particle\+Collection\+\_\+t} object. This is due to both the aforementioned py\+R\+O\+OT and C++ R\+O\+OT compatibility issues alongside the usage of syntactic sugar to allow for easier columnar data entry.\hypertarget{SimpleParticleCollection_t_collection_initializer_from_column_doc}{}\doxysection{collection\+\_\+initializer\+\_\+from\+\_\+column}\label{SimpleParticleCollection_t_collection_initializer_from_column_doc}
This initializes a Simple\+Particle\+Collection\+\_\+t from a list of P\+DG ids, and either a set of $P_x$, $P_y$, $P_z$, $E$ {\itshape OR} $P_t$, $\eta$, $\phi$, mass depending on the boolean flag set (just like in \mbox{\hyperlink{group__Constructors_particle_initializer}{particle\+\_\+initializer}} particle\+\_\+initializer), and is documented \mbox{\hyperlink{group__Constructors_collection_initializer_from_column}{here}}.
